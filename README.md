# signal-desk-web

`SignalDesk` 프론트엔드(React + Vite).

## 1차 릴리즈 범위
- `시장` 탭
- 한국/미국 분리
- 한국/미국 장 상태
- 정규장/장전/장후/휴장
- 지수 차트(일별/월별/연별)
- 캔들 + 거래량 + 이동평균선(5/20/60)
- 수급(개인·외국인·기관)
- 공포/열기 지표
- 뉴스 군집화 + 시장 요약
- `종목` 탭
- 대표 종목 + 검색
- 종목 상세 패널(연관 뉴스/관심/보유/AI 추천 상태)
- 관심종목 직접 추가/삭제
- `포트폴리오` 탭
- 보유 종목 손익/평가
- 포트폴리오 직접 추가/삭제
- `AI 추천` 탭
- 추천 종목 + 추천 성과
- 추천 종목과 연관 뉴스 연결
- 추천 근거/성과 로그 타임라인
- `모의투자` 탭
- 모의 보유종목/거래 로그 직접 관리

## 제품 포지션
- `signal-desk-web`은 SignalDesk의 메인 사용 화면이다.
- 시장, 종목, 포트폴리오, AI 추천, 모의투자까지 전체 기능을 웹에서 모두 다룬다.
- 모바일 앱은 빠른 확인용이고, 직접 입력/관리 흐름은 현재 웹이 중심이다.

## 현재 UI 구조
- 상단 개요 카드 + 시장 요약
- `시장`, `종목`, `포트폴리오`, `AI 추천`, `모의투자` 탭 구성
- 시장 탭에서 한국/미국 전환, 지수/차트, 수급, 공포/열기 지표, 뉴스 요약을 함께 제공
- 홈 화면에서 `관심종목 이상징후` 카드로 가격 급등락, 뉴스 집중, AI 추천 정렬, 보유 손익 경고를 우선 노출
- 종목 탭에서 검색, 종목 상세 패널, 관심종목/보유/AI 추천 상태를 한 화면에서 확인
- 포트폴리오, AI 추천, 모의투자 탭에서 직접 저장/삭제 작업 지원

## 코드 구조
- `App.tsx`: 화면 상태, 파생 데이터 계산, 탭 조합
- `components/tabs/*`: 시장/종목/포트폴리오/AI/모의투자 탭 렌더링
- `components/MarketIndexCharts.tsx`: 차트 카드 렌더링
- `hooks/useMarketData.ts`: 시장/탭 데이터 조회와 탭별 로딩 상태
- `hooks/useWorkspaceActions.ts`: 관심종목/포트폴리오/AI/모의투자 CRUD
- `lib/format.ts`, `lib/news.ts`: 포맷/뉴스 분류 유틸
- `types.ts`, `constants.ts`: 타입과 기본값 정의

## 성능 전략
- 초기 진입은 `summary`, `sections`, `news`만 호출
- 나머지 탭은 전환 시 지연 로딩
- 저장/삭제 후 해당 탭 데이터만 재호출
- 차트 모듈은 시장 탭 `지수와 차트` 진입 시점에 lazy import
- 차트는 `MarketIndexCharts` 청크로 분리되어 초기 로딩과 분리
- 초기 화면에서 차트 엔진을 바로 올리지 않고, 시장 탭 차트 영역 진입 시점에만 불러온다
- 탭별 에러/로딩 상태를 분리해 한 탭 실패가 전체 실패처럼 보이지 않게 처리
- 차트 엔진은 `lightweight-charts` 기반으로 교체해 캔들/거래량 렌더링 품질과 번들 크기를 함께 줄였다

## API 연동 범위
- `summary`, `sections`, `news`로 초기 시장 화면 구성
- `watchlist`, `portfolio`, `ai-recommendations`, `paper-trading`는 탭 진입 시 추가 호출
- 저장/삭제 작업은 `/api/v1/workspace/*` 경로를 사용

## 실행
```bash
npm install
npm run dev
```

- 로컬 개발 서버에서 API 직결 시

- Docker(`signal-desk-web` nginx 프록시) 사용 시 기본값(`/api`)으로 동작

- web: `http://localhost:4180`

## 참고
- 현재 화면 구성은 단일 앱 셸 중심으로 빠르게 발전시키는 구조다.
- 이후 탭/도메인별 컴포넌트 분리와 디자인 시스템 정리는 추가 고도화 대상이다.
- 차트는 `lightweight-charts` 기반으로 교체해 기존 ECharts 대비 번들을 더 가볍게 유지한다.
